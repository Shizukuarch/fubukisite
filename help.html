Sitemap

  • Key - <> = required input | () = optional input
  • /help - Gives a link to this document
  • /invite - Gives a link to invite the bot.
  • /changelog - Gives a link to the changelog
  • /userinfo <user> - Displays information about a user
  • /slowmode <time> (channel) (reason) - Sets slowmode on a specified channel (Default current channel)
  • /swslowmode <time> - Sets slowmode for the entire server (requires administrator permission)
  • /kick <user> (reason) - Kicks a user from the server
  • /ban <user> (reason) - Bans a user from the server and clears their last 7 days of messages
  • /guildinfo - Displays information about the server
  • /clear <amount> (from user) - Clears the specified amount of messages from the server
  • /unban <user> (reason) - Unbans a user from the server
  • /tempban <user> <time> <unitsoftime> (reason) - Temporarily bans a user from the server
  • /config - Use with no arguments for current config. Options for arguments listed below.
  • /cmdperms - <command> <tier> - Restricts the command usage to the specified permission levels. More info below.
  • /modrole - <command> <tier> - Gives this role the specified permission level. More info below.
  • /admin - <args> - No need to worry about this one. :)
  • /softban <user> (reason) - Bans a user from the server and clears their last 7 days of messages, and then immedietly unbans them. Effectively a kick + clear messages.
  • /recycle - (channel) - Deletes a channel and re-creates it to clear all messages.
  • /support - Allows servers to enable or disable support mode to get support from Fubuki Support Staff.
  • /buttoncode - <type> <role> <color> <title> - Generates a code to make a role button.
  • /sendbuttonmsg - <text> <button1> (button2) (button3) (button4) (button5) - Sends a message containing a button.
  • /timeout - <user> <time> <units> (reason) - Prevents a user from sending messages, adding reactons, or joining voice channels. Up to 28 days.
  • /untimeout - <user> (reason) - Removes a user's timeout.
    The permission levels can be assinged to command and allow management of who can use what commands. Perm 0 is default and just looks at basic server permissions and role hierarchy. 1 is the first level that can be assinged, while it can be granted custom access to commands, it isn't considered a moderator permission level by the bot. 2 and 3 are like 1, but are cosidered moderator levels. 4 is automatically assinged to administrators. 5 is automatically assinged to the owner. Scenerio: Someone with level 3 cannot ban someone with level 3 but can ban anyone with a lower level. Likewise anyone with a lower level of permission couldn't ban someone with a level higher than theirs. Meaning the server owner can ban anyone that the bot can. Commands that have been restricted can be used by anyone with a level higher than or equal to the commands restriction level, meaning that administrators and the owner can always use any command.
    Config basics
  • Welcome Role - The role that will be assinged to users after they have lost the pending flag. (Meaning this works with membership screening!)
  • Welcome Message - The message that will be sent when a new member joins the server. (Assuming the welcome channel is set up.)
  • Welcome Channel - The channel that the welcome message will be sent to when a new member joins the server. (Assuming the welcome message is set up.)
  • Hide Owner - If the owner should be hidden from the userinfo and guildinfo commands. (Only can be used by the server owner.)
  • Hide Mod Actions - If moderation actions should appear in chat or only be posted in the audit log.
  • Reset - WARNING - This fully deletes all of your server's config from the database and resets it to default. As if you had just invited the bot. You must paste your server ID in the field to confirm. (Only can be used by the server owner.)
